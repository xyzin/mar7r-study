==========================================================================
    Visual Studio Team System: 테스트 작성 및 실행 개요        
==========================================================================

이 개요에서는 Visual Studio Team System 및 Visual Studio Team Edition for 
Software Testers에서 테스트를 작성하고 실행하는 기능에 대해 설명합니다.

테스트 열기
-------------
테스트를 열려면 테스트 프로젝트 또는 테스트 정의가 들어 있는 테스트
메타데이터 파일(확장명 .vsmdi)을 엽니다. 솔루션 탐색기에서 테스트
프로젝트 및 메타데이터 파일을 찾을 수 있습니다.

테스트 보기
-------------
사용할 수 있는 테스트를 보려면 [테스트 뷰] 창을 엽니다. Team Edition for Software Testers를
설치한 경우에는 [테스트 목록 편집기] 창을 열어
테스트를 볼 수도 있습니다.

[테스트 뷰] 창을 열려면 [테스트] 메뉴를 클릭하고 [창]을 가리킨 다음
[테스트 뷰]를 클릭합니다. Team Edition for Software Testers를 설치한 경우
[테스트 목록 편집기] 창을 열려면 [테스트]를 클릭하고 [창]을 가리킨 다음
[테스트 목록 편집기]를 클릭합니다.

테스트 실행
-------------
[테스트 뷰] 창 및 [테스트 목록 편집기] 창에서 테스트를 실행할 수 있습니다.
이러한 창을 여는 방법은 테스트 보기를 참조하십시오. [테스트 뷰] 창에 표시된
테스트를 하나 이상 실행하려면 먼저 창에서 원하는 테스트를 선택합니다.
Shift 또는 Ctrl 키를 누른 상태에서 클릭하면 테스트를 여러 개 선택할
수 있습니다. 그런 다음 [테스트 뷰] 창 도구 모음에 있는 [테스트 실행] 단추를
클릭합니다.

Visual Studio Team Edition for Software Testers를 설치한 경우에는
[테스트 목록 편집기] 창을 사용하여 테스트를 실행할 수도 있습니다. [테스트 목록 편집기]에서 테스트를 실행하려면
실행할 각 테스트 옆에 있는 확인란을 선택한 다음 [테스트 목록 편집기] 창 도구 모음에 있는
[테스트 실행] 단추를 클릭합니다.

테스트 결과 보기
--------------------
테스트를 하나 이상 실행하면 [테스트 결과] 창에 테스트 실행 결과가
표시됩니다. 실행 중인 각 테스트는 해당 상태를 볼 수 있도록 별도의 줄에
표시됩니다. 이 창의 위쪽에는 상태 표시줄이 있고 여기에는
전체 테스트의 실행에 대한
요약 정보가 표시됩니다.

특정 테스트 결과에 대한 세부 결과를 보려면 [테스트 결과] 창에서 원하는
테스트를 두 번 클릭합니다. 그러면 테스트에서 반환한 특정 오류
메시지 등 특정 테스트 결과에 대한 자세한 정보를 제공하는 창이
열립니다.

테스트 실행 방법 변경
-----------------------------------
테스트를 하나 이상 실행할 때마다 테스트 실행 방법을 결정하는 일련의 설정이
사용됩니다. 이러한 설정은 “테스트 실행 구성” 파일에
들어 있습니다.

테스트 실행 구성 파일에서 변경할 수 있는 사항으로는 다음과 같은 것이
있습니다.

- 각 테스트 실행의 이름 지정 체계 변경
 - 테스트를 원격으로 실행할 수 있도록 테스트가 실행되는
   테스트 컨트롤러 변경
 - 테스트에 의해 어느 코드 줄이 검사되는지 확인할 수 있도록
   테스트 중인 코드에 대한 코드 검사 데이터 수집
 - 테스트 배포 활성화/비활성화 
 - 테스트를 실행하기 전에 배포할 추가 파일 지정
 - ASP.NET 단위 테스트를 실행할 다른 호스트(ASP.NET) 선택
 - 스마트 장치 단위 테스트를 실행하기 위해 다른 호스트인 스마트 장치 테스트 호스트 선택
 - 테스트를 실행하는 테스트 에이전트에 대해 다양한 속성 설정
 - 테스트를 실행할 때마다 필요한 테스트 환경을 정확히 설정할 수 있도록
   각 테스트의 시작 및 종료 시 사용자 지정 스크립트 실행
 -테스트 및 테스트 실행에 대해 시간 제한 설정
 - 브라우저 조합 및 테스트 실행 시 웹 테스트를 반복할 횟수
   설정

기본적으로 테스트 실행 구성 파일은 새 테스트 프로젝트를 만들 때마다
만들어집니다. 솔루션 탐색기에서 이 파일을 두 번 클릭한 후 해당 설정을
수정하여 파일의 내용을 변경할 수 있습니다. 테스트 실행 구성 파일의
확장명은 .testrunconfig입니다.

솔루션에는 테스트 실행 구성 파일이 여러 개 포함될 수 있지만 이러한
파일 중에서 하나의 파일만 테스트 실행에 현재 사용되는 설정을 결정합니다.
이 파일을 “활성” 테스트 실행 구성 파일이라고 합니다. 활성 테스트 실행 구성 파일은 [테스트]
메뉴에서 [활성 테스트 실행 구성 선택]을 클릭하여 선택할 수 
있습니다.

테스트 종류
----------
Visual Studio Team Edition for Software Testers를 사용하면 다음과
같이 여러 종류의 테스트를 만들 수 있습니다.

단위 테스트: C++, Visual C# 또는 Visual Basic에서 소스 코드를 실행하는
프로그래밍 방식의 테스트를 만들려면 단위 테스트를 사용합니다. 단위 테스트는 적절한 매개 변수를 전달하여
클래스의 메서드를 호출하고, 반환된 값이 예상했던 값인지
확인합니다.
단위 테스트에는 다음과 같은 세 종류의 테스트가 있습니다.
 - 데이터 기반 단위 테스트는 데이터 소스의 각 행에 대해 반복적으로 호출되는
   단위 테스트를 구성할 경우 만들어집니다. 각 행의 데이터는
   단위 테스트에서 입력 데이터로 사용됩니다.
 - ASP.NET 단위 테스트는 ASP.NET 웹 응용 프로그램의 코드를 실행하는
   단위 테스트입니다.
 - 스마트 장치 단위 테스트는 스마트 장치 또는 에뮬레이터에 배포되어
   스마트 장치 테스트 호스트에 의해 실행되는 단위 테스트입니다.  

웹 테스트: 웹 테스트는 Microsoft Internet Explorer를 사용하여 브라우저 세션에서 기록하는
순서가 지정된 일련의 HTTP 요청으로 구성됩니다. 특정 페이지에
지정한 문자열이 포함되어 있는지 여부와 같이
요청한 페이지 또는 사이트에 대한 세부 정보를 확인할 수 있습니다.

부하 테스트: 부하 테스트를 사용하여 단위, 웹 및 제네릭 테스트와 같이 사용자가 직접 실행하지 않는 테스트를 캡슐화하고 가상 사용자를 사용하여 동시에 이러한 테스트를 실행할 수 있습니다. 이러한 테스트를 부하 상태에서 실행하면
성능 및 기타 카운터를 비롯하여 테스트 결과가 테이블 및 그래프로 생성됩니다.

제네릭 테스트: 제네릭 테스트는 Visual Studio에서 테스트로 기능하도록
래핑된 기존 프로그램입니다. 다음은 제네릭 테스트로 변환할 수 있는
테스트 또는 프로그램의 예입니다.
 - 프로세스 종료 코드를 사용하여 테스트 성공 여부를 전달하는
   기존 테스트. 0은 성공했음을 나타내고 그 외의 값은
   실패했음을 나타냅니다.
 - 테스트 시나리오에서 특정 기능을 가져오는 일반 프로그램
 - “요약 결과 파일”이라고 하는 특수 XML 파일을 사용하여 세부 결과를
   전달하는 테스트 또는 프로그램

수동 테스트: 이 테스트 종류는 자동화된 스크립트를 사용하지 않고 테스트 엔지니어가 직접 테스트 작업을 완료해야 하는 경우에 사용됩니다.

순서가 지정된 테스트: 순서가 지정된 테스트는 사용자가 지정한 순서대로
일련의 테스트를 실행하기 위해 사용됩니다.
